Customer Management System
מערכת לניהול לקוחות שנבנתה כמיני-פרויקט במסגרת הכנה לפרקטיקום. המערכת מאפשרת צפייה, הוספה, עדכון וניהול מרוכז של לקוחות תוך שימוש בארכיטקטורת 3 שכבות.

1. נושא הפרויקט
הנושא שנבחר הוא ניהול לקוחות (CRM).
המערכת משמשת ככלי פנימי לניהול פרטי הלקוחות של העסק, כולל קישור למצב (סטטוס) הלקוח, עיר המגורים שלו ותיעוד פעילויות.

2. תיאור הטבלאות והקשרים
מסד הנתונים מורכב מ-4 טבלאות. טבלת Customers היא הטבלה הראשית, ושאר הטבלאות מקושרות אליה:

Customers (לקוחות): טבלת הליבה של הפרויקט. מכילה את הפרטים האישיים של הלקוח (שם מלא, טלפון, אימייל, הערות ותאריך יצירה).

מכילה מפתח זר (StatusId) לטבלת CustomerStatuses.

מכילה מפתח זר (CityId) לטבלת Cities.

CustomerStatuses (סטטוס לקוחות): טבלת עזר המכילה רשימת ערכים לסטטוס הלקוח (למשל: פעיל, לא פעיל).

Cities (ערים): טבלת עזר המכילה את רשימת הערים האפשריות לשיוך הלקוח.

CustomerActivities (פעילויות לקוחות): טבלה המתעדת את היסטוריית הפעולות עבור כל לקוח. מקושרת לטבלת Customers ביחס של "רבים לאחד" עם הגדרת מחיקה מדורגת (ON DELETE CASCADE).

3. רשימת ה-Stored Procedures
הפרויקט לא משתמש בשאילתות SQL דינאמיות ב-API. כל האינטראקציה מול מסד הנתונים מתבצעת דרך 9 ה-Stored Procedures הבאות:

ניהול לקוחות (CRUD ראשי):

Customers_Create: הוספת לקוח חדש. מקבלת פרמטרים ומחזירה את מזהה הלקוח החדש.

Customers_Update: עדכון פרטי לקוח קיים לפי מזהה.

Customers_GetById: שליפת נתוני לקוח ספציפי לפי מזהה. (מבצעת JOIN כדי להחזיר שמות ידידותיים של העיר והסטטוס). בנוסף, מחזירה סט תוצאות שני הכולל את רשימת הפעילויות של אותו לקוח.

Customers_GetAll: שליפת רשימת כל הלקוחות. כוללת תמיכה בחיפוש טקסט חופשי (פרמטר @Search) לסינון לפי שם הלקוח.

Customers_Delete: מחיקת לקוח קיים לפי מזהה.

Customers_ChangeStatus: פעולה ייעודית לעדכון מהיר של סטטוס הלקוח.

שליפת נתוני עזר (עבור טפסים):
7.  CustomerStatuses_GetAll: שליפת רשימת הסטטוסים האפשריים.
8.  Cities_GetAll: שליפת רשימת הערים.

ניהול פעילויות:
9.  CustomerActivities_Create: הוספת תיעוד פעילות חדשה ללקוח ספציפי.

4. הוראות הרצה (צד שרת וצד לקוח)
דרישות קדם
התקנת SQL Server (גרסת Developer/Express) ו-SSMS.

התקנת .NET SDK (גרסה 8/9).

התקנת Node.js ו-Angular CLI.

הקמת מסד הנתונים
פתחו את ה-SQL Server Management Studio (SSMS).

צרו מסד נתונים חדש.

הריצו את סקריפט ה-SQL המצורף בתיקיית DB כדי ליצור את כל הטבלאות וה-Stored Procedures.

הפעלת השרת (API)
פתחו את תיקיית צד השרת (api / פרויקט ה-.NET) ב-Visual Studio או VS Code.

פתחו את הקובץ appsettings.json ועדכנו את מחרוזת ההתחברות (DefaultConnection) כך שתצביע למסד הנתונים המקומי שלכם.

הריצו את הפרויקט (באמצעות כפתור ההפעלה ב-Visual Studio או הרצת הפקודה dotnet run בטרמינל). ה-Swagger אמור להיפתח בדפדפן.

הפעלת צד הלקוח (Angular)
פתחו את טרמינל מערכת ההפעלה בתוך תיקיית צד הלקוח (client).

הריצו את הפקודה npm install כדי להתקין את כל הספריות הדרושות.

ודאו שהכתובת של ה-API בקובץ data.service.ts תואמת לפורט שעליו רץ השרת שלכם ב-.NET.

הריצו את האפליקציה עם הפקודה ng serve.

פתחו את הדפדפן בכתובת: http://localhost:4200.
